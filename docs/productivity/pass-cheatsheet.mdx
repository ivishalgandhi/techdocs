### Initial Setup

  <code>gpg --full-generate-key</code>

</div>

<div className="command-item">

  <div className="command-desc">Initialize password store with GPG key</div>

  <div className="command-desc">Initialize git in password store</div>

### Adding Passwords

<div className="command-item">
  <code>pass insert path/to/password</code>
  <div className="command-desc">Add a new password (prompted to enter)</div>
</div>

<div className="command-item">

</div>

<div className="command-item">

  <div className="command-desc">Generate random 24-character password</div>
</div>

<div className="command-item">
  <code>pass generate -n path/to/password 20</code>

</div>

### Important Locations

<div className="command-item">
  <code>~/.password-store/</code>
  <div className="command-desc">Default password store directory</div>
</div>

<div className="command-item">
  <code>~/.gnupg/</code>
  <div className="command-desc">GPG configuration and keys directory</div>
</div>

<div className="command-item">
  <code>~/.password-store/.gpg-id</code>
  <div className="command-desc">File containing GPG key ID used for encryption</div>
</div>

<div className="command-item">
  <code>/usr/share/bash-completion/completions/pass</code>
  <div className="command-desc">Bash completion script (source in .bashrc)</div>
</div>

</div>

</div>

:::warning Security Reminder
Always use a **private Git repository** for your password store, even though passwords are encrypted. Never commit your GPG private keys to the repository.
:::

:::tip Pro Tip
Use `pass -c` instead of `pass show` to avoid displaying passwords on screen. The clipboard auto-clears after 45 seconds for security.
:::
